⬡ Score ▦ Fields ✔ Auto-tests
Score
Overview KPI cards▼

The four top cards aggregate counts across all non-excluded projects ("Complience test" is excluded from Score calculations).

  • Target Fields number by 31 Dec, 2026 — count of all included rows (or the selected Priority subset).
  • To Do — fields whose current status is To Do (computed status by default, model-specific status when a model row is selected).
  • In Progress — fields whose computed status is In Progress.
  • Ready for arbitrage — fields where at least 2 of 4 model statuses are Done.

KPI counts update when either a Priority row or a Model row is selected.

Field status definition▼

Each field row has four model statuses: Internal Tool, GPT, Gemini, Claude. The computed status for a field is determined by this priority order:

  • Done — 3 or more of the 4 model columns have status Done.
  • In Progress — at least 1 model column has the literal status In Progress (and the Done threshold is not met).
  • To Do — all other cases.

Status matching is case-insensitive. in_progress is treated the same as In Progress.

Fields by Priority table▼

Rows show totals broken down by Priority (P1–P5) for the active model or overall. Clicking a priority row filters both the KPI cards and the Model Performance table to that priority.

Click the same row again to clear the filter. The × clear link in the section header also resets both filters.

Model Performance table▼

Each row represents one model (Internal Tool, GPT, Gemini, Claude). Counts reflect status of the model-specific column on each field row.

Clicking a model row cross-filters the Priority table to show only that model's data. Both filters are mutually exclusive: selecting a model clears a priority filter and vice versa.

Weekly Dynamics panel▼

Snapshots are taken automatically every Friday at 9:00 PM ET and stored in snapshots.json. Manual snapshots can be triggered with the + Snapshot button.

  • Start row — optional baseline snapshot (week_num = -1). Shows absolute totals for each model's Done / In Progress columns (not deltas).
  • W0, W1, W2 … — weekly snapshots. Counts shown are deltas vs the previous snapshot. Zero deltas are displayed as –.
  • Done Δ — number of fields that moved into Done status since the previous snapshot.
  • Prog Δ — change in In Progress count.
  • New Δ — new field rows added to the sheet since the previous snapshot.
  • Removed Δ — field rows removed from the sheet since the previous snapshot.

Week numbers are calculated relative to the baseline Sunday Mar 15, 2026. Week 0 covers Mar 15–21.

Positive values are highlighted with colored backgrounds (green = Done, amber = In Progress, blue = New).

Excluded projects▼

The project "Complience test" is excluded from Score KPI cards, Fields by Priority, and Model Performance calculations.

In Weekly Dynamics, compliance rows are excluded from Start totals and W0 Total/Done logic.

Its fields still appear on the Fields page when no project filter is active.

Fields
Overview cards (90% coverage rates)▼

Five KPI cards display the share of rows that reached 90% or higher in the currently filtered set:

  • Internal Tool % — percent of all filtered rows where Internal tools % >= 90 (includes "Complience test").
  • GPT % — percent of filtered rows where GPT % >= 90, excluding project "Complience test" from denominator.
  • Gemini % — percent of filtered rows where Gemini % >= 90, excluding project "Complience test".
  • Claude % — percent of filtered rows where Claude % >= 90, excluding project "Complience test".
  • Goal — All fields with 90%+ rate — percent of all filtered rows where at least one of Internal/GPT/Gemini/Claude percentages is >= 90.

When a denominator is empty, the card displays —.

Fields table columns▼

The table has 14 columns. Key derived columns:

  • Goal — maximum completion % across all four models for that row: MAX(Int%, GPT%, Gemini%, Claude%).
  • Int. % — raw Internal tools % from the Excel sheet.
  • GPT %, Gemini %, Claude % — raw model % columns from the Excel sheet.
  • Internal Tool / GPT / Gemini / Claude — raw status values rendered as badges.
  • Priority — P1 through P5 badge from the Priority column.
Filters▼

Six independent filters are applied in sequence (AND logic):

  • Project — multi-select.
  • Category — multi-select.
  • Priority — multi-select P1–P5.
  • Status — multi-select values: Done / In Progress / To Do.
  • Model — multi-select model columns used only together with Status (for example: Status=Done + Model=GPT means GPT status must be Done).
  • Search — free-text match against Document type and Field name columns (case-insensitive).

Without a Status filter, Model selection alone does not reduce the row set.

The Overview cards always reflect the currently filtered rows.

Sorting▼

Click any column header to sort ascending. Click again to sort descending. An indicator arrow (▲/▼) marks the active sort column.

Default sort is the original row order from the Excel sheet.

Auto-tests
Overview card definitions▼
  • Total Documents — count of all auto-test rows matching active project/category/search filters.
  • 100 Docs Set — rows where the 100 docs set column equals Yes.
  • To Do — rows with status To Do (or blank/unrecognized).
  • Revalidated — rows with status Revalidated.
  • Done — rows with status Done.

Overview card counts are based on the full project/category/search-filtered set, combined with any active priority row selection (but independent of status card selection).

Clicking a card (except Total) filters the detail table to that status / property.

Status definitions▼
  • Done — status column value is done (case-insensitive).
  • Revalidated — status column value is revalidated (case-insensitive). Shown with a blue badge.
  • In Progress — status column value is in progress or in_progress (case-insensitive).
  • To Do — all other values, including blank.

The KPI cards and card-based status filter include Done, Revalidated, and To Do. In Progress appears in the table badge but has no dedicated KPI card/filter.

Documents by Priority table▼

Shows a breakdown of auto-test documents by priority level. Columns: Priority | Total | 100 Docs Set | To Do | Revalidated | Done.

Priority table counts are always based on the base-filtered rows (project/category/search) — the selected status card does NOT affect priority table counts, only the detail table.

Clicking a priority row filters the detail table AND the Overview cards to that priority only. Click the same row again to clear.

Changing a project/category/search filter resets the priority selection automatically.

Combined filtering logic▼

Filters are applied in this order to build the final detail table rows:

base rows → project filter → category filter → search filter → priority filter (from row click) → status filter (from card click)

The Overview cards reflect after-priority, before-status filtering (so the card counts always sum to Total).

Target number vs Actual number▼

Target number — the expected number of test examples for a document type, read from the Target number column in the Autotests sheet.

Actual number — the current number of test examples, read from the Actual number column.

These values are displayed for visibility and can be used for manual analysis; they do not currently change status classification in calculations.

100 Docs Set▼

100 Docs Set is a flag column (Yes / No) indicating that a representative set of 100 documents has been assembled for this document type.

It is independent of the Status column — a document type can be 100-docs-set without being Done, and vice versa.

Data source▼

Auto-tests data is read from the Autotests sheet in data.xlsm via the /api/autotests endpoint. The sheet columns used are:

  • Project, Category, Document type, Priority, Status
  • 100 docs set, Actual number, Target number

All unnamed/empty columns are dropped. Data is refreshed on page load and when the Refresh button is clicked.
